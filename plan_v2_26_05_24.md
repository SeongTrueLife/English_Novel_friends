# Plan v2 — 2026-05-24

SentenceMate Reader 개선 계획. 사용자와의 브레인스토밍 결과를 정리한 문서.

---

## Context

현재 앱을 사용하면서 사용자가 느낀 두 가지 핵심 한계:

1. **단어장 구조 문제** — AI 응답에서 단어와 설명을 함께 드래그해서 한 덩어리로 저장하는 방식이라 "외울 단어"와 "설명"이 구분되지 않음. 책 원문 문장이 앵커로 같이 저장되지 않아 암기 효율도 떨어짐.
2. **AI 프롬프트(3단계 구조)의 한계** —
   - 1단계 직독직해가 추론력을 키우기보다 한국어 매핑에 의존하게 만듦
   - 3단계가 가장 효과적이지만 "해당될 때만 선별" 규칙으로 사용자가 궁금한 단어 누락
   - 영어 본질에 가까운 문법 설명(시제·관사·전치사 등) 부족

**목적**: 추론력을 살리는 학습 사이클로 응답 구조를 재설계하고, 단어장을 단어/설명/예문(원문) 분리된 카드 형태로 만들며, AI 응답에서 단어 카드 자동 생성 가능하도록 한다.

---

## 결정 사항

### A. 응답 구조 (프롬프트 재설계)
- **직독직해(구 1단계) 완전 제거**
- **새 1단계 — [핵심 단어] + [영어식 사고]** (기본 펼침)
  - [핵심 단어]: 선택 문장의 중급+ 단어를 빠짐없이 (단어 누락 안전망)
  - [영어식 사고]: 한국식 5형식 회피, 영어식 카테고리 — 시제·관사·전치사·가산/불가산·명사화·문화·작가 의도 등 해당될 때만
- **새 2단계 — [자연 해석]** (기본 접힘, 탭하면 펼침)
  - 추론 사이클: 사용자가 단어 + 어순으로 먼저 추측 → 탭으로 정답 확인
- 시스템 프롬프트는 zero에서 재작성. 친구처럼 반말 톤은 유지.

### B. 출력 형식 — JSON
- Gemini Structured Output (`responseMimeType: "application/json"` + `responseSchema`) + 시스템 프롬프트 형식 명시, 둘 다 사용
- 스트리밍 포기 trade-off 수용 (응답 완료까지 1~3초 빈 상태)
- 잠정 스키마:
```json
{
  "vocab": [
    { "word": "...", "meaning": "...", "note": "..." }
  ],
  "thinking": [
    { "type": "grammar|core_image|culture|author_intent|...", "title": "...", "body": "..." }
  ],
  "naturalTranslation": "..."
}
```

### C. 사용자 추가 프롬프트
- 시스템 프롬프트는 항상 전송. 사용자가 입력하면 `buildUserMessage`에 `[사용자 추가 요청]` 한 줄 추가.
- 입력 UI = **B + C 방식 결합**:
  - **(B)** Ask AI 옆 `+` 토글 → 평소 즉시 전송, `+` 누르면 textarea(2~3줄 자동 확장) 펼침
  - **(C)** 응답 시트 안 follow-up 입력란 → **채팅형 누적(C-1)**
- follow-up은 multi-turn (Gemini `contents` 배열에 이전 대화 누적)
- 다른 문장 선택 시 새 세션 시작 (토큰 폭증 방지, 3~5턴 이내 예상)
- 프리셋 칩은 MVP 제외 (실사용 데이터 모인 후 추후 추가)
- 사용자 입력 내역 `localStorage`에 누적(`userPromptHistory: []`) — 추후 패턴 분석용

### D. 단어장 (Wordbook)
- 새 데이터 모델:
```js
{
  id, word, meaning,
  exampleSentence,         // 책 원문 문장 (앵커 — 암기 효율)
  aiExplanation,           // AI가 준 설명 (note)
  bookTitle, chapter, createdAt,
  // SRS용 필드 (추후 단계)
  reviewCount, lastReviewedAt, nextReviewAt, easeFactor
}
```
- 카드 추가 UX = **A4 방식**:
  - AI 응답의 `vocab` 배열을 칩/리스트로 표시
  - 각 단어 옆 `+` 버튼 → 클릭으로 카드 즉시 추가 (책 원문 자동 첨부)
- 기존 단어장 데이터 마이그레이션 정책: **별도 결정 필요**

### E. 모델
- **gemini-3-flash 유지** — 가성비 + JSON Structured Output 지원

### F. 디바이스 우선순위
- 1차: **태블릿**, 2차: **스마트폰**
- 모든 UI 결정은 이 기준 — 한 손 조작, 키보드 올라옴 대응

- [VocabList.jsx](sentencemate-reader/src/components/VocabList.jsx) — 새 필드(meaning, exampleSentence) 분리 표시
