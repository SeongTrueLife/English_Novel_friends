## 0. 한 줄 요약

epub을 읽다가 문장/단어를 선택하면 AI가 **JSON으로** 풀이를 주고, 사용자가 단어/문법 **카드**로 저장해 **플래시카드**로 복습하는 "읽으며 배우는" 리더. **익명 시작 → 이메일 연결**로 기기 간 동기화. **PWA.**

## 2. 디자인 원칙 — "조용한 서재 (quiet study)"

1. **읽기 우선 · 추론 우선 · 절제.** 화려함 배제. 정보를 쏟지 않고 "한 박자 참기"를 유도.
2. **타이포로 역할 분리** — 본문 = 세리프, UI = 산세리프. ("이건 읽기 앱"이라는 선언)
3. **그림자 대신 0.5px hairline.** 평평한 표면.
4. **악센트는 한 색(청록)** — 선택·저장·핵심 한 줄·활성 등 의미 있는 순간에만. 무지개 금지.
5. **순백/순흑 회피** — 따뜻한 미색 배경 + 잉크빛 회색 글자.
6. **시그니처 디테일** — 본문에서 고른 문장 ↔ 오른쪽/아래 풀이가 **같은 악센트로 연결**.

## 4. 반응형 규칙 — 방향(orientation) 기준

> **세로면 아래, 가로면 옆.** 폭(px)이 아니라 **기기를 쥔 방향**으로 가른다.

| 요소              | portrait (세로 — 기본)         | landscape (가로)         |
| ----------------- | ------------------------------ | ------------------------ |
| AI 풀이 표면      | **바텀시트**(본문 위로 떠오름) | **사이드 패널**(본문 옆) |
| 내비게이션        | **하단 탭바**                  | **좌측 레일**            |
| 라이브러리 그리드 | 3열                            | 4–5열                    |

- **브레이크포인트 = `orientation`.** `@media (orientation: portrait/landscape)` 또는 JS `matchMedia`. (풀이 표면처럼 컴포넌트 자체가 갈리는 곳은 `matchMedia` + state.)
  - **왜 폭이 아니라 방향인가**: 태블릿 세로 폭은 768~834px라, 폭 기준이면 '넓음'으로 **오분류**돼 세로로 읽는데 사이드패널이 떠버린다(1차 기기·기본 자세에서 규칙이 거꾸로 작동). 방향은 곧 사용자 자세 = 의도라 우회 없이 일치.
- **회전 시 상태 보존**: 읽다가 기기를 돌리면 바텀시트↔사이드패널은 **레이아웃만** 전환되고, 열림 여부·응답 내용·follow-up·스크롤 위치는 그대로 유지.
- **태블릿 세로 ≠ 폰 세로 (reading measure)**: 태블릿 세로는 폰의 2배 폭이라 본문을 꽉 채우면 한 줄이 너무 길어 가독성이 무너진다. 세로라도 **본문은 `max-width ~680px` 가운데 정렬**(양옆 여백), 바텀시트도 그 폭에 맞춰 가운데(전체 폭 X). (§3 토큰의 680px 정신 유지.)
- **폰 가로 엣지**: 폰을 가로로 돌리면 landscape라 규칙상 사이드패널이 되는데, 폰 가로는 높이가 낮아(~390px) 풀이 영역이 답답할 수 있다. 드문 사용이라 **MVP는 landscape 규칙 그대로 적용**, 불편이 실측되면 그때 `min-height` 가드로 폰 가로만 바텀시트 fallback (지금은 과한 설계라 보류).
- **리더(읽기 모드)에선 내비를 숨긴다**(몰입). 다른 모든 최상위 화면엔 내비 표시.

## 6. 화면 명세

### 6.1 라이브러리 (BookLibrary) — 앱 첫 화면

- **구성**: 상단 고정(검색·정렬) → "계속 읽기"(현재 책 크게 + 진행률 + "이어 읽기") → "내 서재" 표지 그리드(세로 3열, 진행률 바, 카운트) → 점선 "책 추가" 타일.
- **표지**: epub에서 렌더, 없으면 **타이포그래피 폴백**(세리프 제목+저자, 상단 악센트선). 깨진 이미지 금지.
- **데이터**: `user_books`(progress_cfi, last_opened) + `books`(title, author).
- **상호작용**: 표지 탭 → 리더(이어 읽기) / 추가 타일 → 책 추가 모달.
- 책 많아지면 **세로 스크롤**(상단 계속읽기·검색은 고정).
- 빈 상태 → §6.8.

### 6.2 리더 + AI 응답 (EpubReader + AIResponse) — 앱의 심장

- **본문**: epub.js **paginated**(좌우 쪽넘김), **CFI로 진행 저장**, 세리프, 여백으로 measure 제한, 폰트 크기 조절 버튼.
- **선택 → "Ask AI"**(선택 근처). (추가 프롬프트 `+` 토글은 §10 보류 디테일.)
- **풀이 표면(반응형)**: 세로=바텀시트 / 가로=사이드 패널. 둘 다 **3구역**:
  - 상단 **고정** = 선택한 문장(앵커). 스크롤에 안 쓸려나감.
  - 중앙 **스크롤** = 단어/문법 풀이 + follow-up 누적.
  - 하단 **고정** = ① 자연 해석 토글(아래) + ② "더 물어보기" 입력. **둘 다 채팅 스크롤 밖**이라 follow-up이 아무리 쌓여도 위치 불변.
  - **바텀시트(세로)는 높이 제한**(v1 0.45~0.85 계승) — 시트를 올려도 **본문 몇 줄이 계속 보여** 원문↔풀이 대조 가능(추론-우선). 드래그로 높이 조절.
- **AI 응답 = JSON 파싱 후 구조화 렌더 (raw 출력 금지).** `system_prompt_v3` 스키마.
  - **파싱 3겹**: ① `JSON.parse` → ② **인라인 마크다운 렌더**(`**굵게**`, `\n`) → ③ 컴포넌트 매핑.
  - **굵게(`**...**`)는 항상 "설명 중인 영어 조각"** → 밑줄/하이라이트로 시각 강조.
  - **슬롯 → UI 매핑**:

    | JSON 슬롯                        | UI                                                                                                                          |
    | -------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
    | `vocab[].word`                   | 카드 제목(영어)                                                                                                             |
    | `vocab[].meaning`                | 한 줄 뜻                                                                                                                    |
    | `vocab[].thinking[]`             | **기본 펼침** — 단어 뜻 아래 작은 틴트박스(들여쓰기, 한 단계 낮은 위계). `type`(core_image/culture/author_intent) 작은 태그 |
    | `grammar[].pattern`              | 카드 제목                                                                                                                   |
    | `grammar[].explanation`          | 본문(마크다운)                                                                                                              |
    | `grammar[].interpretation_guide` | **강조된 한 줄**(틴트 박스 + `→`)                                                                                           |
    | `sentence_thinking[]`            | 드묾 — 있을 때만 작은 영역                                                                                                  |
    | `naturalTranslation`             | **하단 고정 토글**(중앙 스크롤 밖). 기본 접힘 — "먼저 추측/적어보기" 유도                                                   |

  - **thinking 기본 펼침의 근거**: thinking은 *정답이 아니라 추론을 돕는 비계(scaffolding)*다 — "consider의 핵심 이미지" 같은 건 영어식으로 추론하는 법을 가르치므로 보여줄수록 추론을 키운다. 반대로 `naturalTranslation`은 *완성된 정답*이라 미리 보면 추론을 대체해버려 **접어서 보호**. (이 펼침/접힘 대비가 추론-우선 정체성의 핵심 표현. plan_v3 A "vocab thinking 펼침 / naturalTranslation 접힘"과 일치.)
  - **빈 슬롯은 섹션째 숨김.** 쉬운 문장은 vocab/grammar 빈 배열 가능. (v1 실사용상 LLM이 전부 빈 응답을 주는 경우는 사실상 없어 별도 "막힘없이 읽혀" 문구는 두지 않음 — 방어적 빈 배열 처리만.)

- **저장**: vocab/grammar 항목마다 우상단 `⊕`. 단어 = word + thinking 통째 1카드 / 문법 = 1카드. **`example_sentence`(원문) + `chapter`는 클라이언트가 자동 첨부.**
  - **피드백**: 낙관적 업데이트 — 누르는 즉시 `⊕→✓`(악센트색, 비활성), 토스트 없음(인라인 ✓로 충분, 절제). **실패 시에만** `✓→⊕` 롤백 + 하단 토스트 "저장 실패 · 다시".
  - **응답 내 중복 차단**: 저장된 항목은 `✓` 유지 + 버튼 비활성 → 같은 응답에서 재저장(더블클릭) 방지. (책 단위 중복 정책은 실사용 후 결정 — db/plan.)
- **follow-up**: 하단 입력 고정, 답변이 중앙 스크롤 영역에 채팅처럼 누적(§10 디테일).
- 상태: 로딩(스켈레톤)·에러(502/429) → §6.8.

### 6.3 단어장 — 목록 모드 (VocabList)

- **역할 = 보관함(archive).** "내가 뭘 모았나"를 훑고 관리(필터·정렬·삭제)하는 곳. **실제 인출 학습은 6.4 학습 모드에서만** 한다 (6.3↔6.4 역할 분리 — 셀프 퀴즈를 양쪽에 두지 않아 "어디서 외우지"가 또렷).
- **탭: 단어 / 문법** (`cards.kind`). **책별 그룹**(섹션 헤더 = 책 + 카운트) + **책 칩 필터** + **정렬(최근 / 가나다)**.
  - **숙어/구동사(idiom·phrasal verb)는 "단어" 탭에 단어와 함께** 다룬다 — 시스템 프롬프트가 숙어를 vocab 항목(`kind='word'`)으로 내보내고, DB에 숙어 전용 kind/플래그가 없어 자동 구분 불가. 즉 분류는 **2분류(단어 탭[단어+숙어] / 문법 탭)**. (숙어만 따로 보려면 향후 `cards`에 플래그 추가 — 다음 단계, 갈아엎기 아님.) **학습 모드(6.4)도 동일**: 단어 세션에 숙어 포함.
- **카드 행**: word(또는 pattern) + 뜻(또는 가이드) + `chapter` 태그 + **원문 예문 펼침**(단어 밑줄). thinking 기본 접힘(목록은 스캔 위주라 응답 시트와 달리 접어둠).
- **선택 모드**: 다중 체크 → **삭제**(RLS `cards_delete_own`) 또는 **선택 학습**(→ 6.4).
- **데이터**: `cards (user_id, book_id, kind, ...)`, 주 쿼리 인덱스 `(user_id, book_id, created_at DESC)`.

### 6.4 단어장 — 학습 모드 (FlashcardStudy + StudyScopeSheet)

- **진입**: "학습 시작" → **범위 프리셋 시트**: 전체 / **챕터별(챕터 칩)** / 최근 N / 아직 안 외운 것 + **섞기** 토글. **단어/문법 세션 분리.**
- **카드**: 앞 = word + 원문 예문(**단어 밑줄 또는 cloze 빈칸**) → 탭하면 **뒤집힘** → 뒤 = 뜻 + thinking. 문법 = pattern → explanation + interpretation_guide.
- **컨트롤**: 이전 / 다음, 진행 바, 섞기. (정답 평가·간격 계산 = SRS, 다음 단계. 이 화면이 그 복습 UI의 그릇이 됨.)

### 6.5 책 추가 (FileUploader + BookMetaForm) — 모달/바텀시트, 3스텝

1. **파일 선택**: 점선 타일/+ → 파일 피커(가로=드래그앤드롭). epub만.
2. **정보 확인/수정**: epub에서 제목·작가·표지 **자동 파싱해 채움(항상 수정 가능)**. **제목 필수 / 작가 선택**. 엉성 파싱이면 **파일명에서 추측 + 경고**. 표지 없으면 타이포 폴백.
   - **hash 재연결**: epub SHA-256 매칭 시 "단어장 N개·진행률 자동 연결" 안내(= **다른 기기 복원**도 이 경로).
3. **저장**: 메타 → Supabase, 파일 → IndexedDB, `user_books` 추가 → 서재 등장(또는 "바로 읽기").

- **에러**: 비-epub/깨짐 → "epub 파일만…", 이미 보유 → "이미 서재에 있어요" + 이동.

### 6.6 통계 — 글로벌 대시보드 (StatsDashboard)

> **MVP는 카드 기반 통계만**(세는 것). 시간/연속일 등 **세션 기반 통계는 다음 단계** — `reading_sessions` 추적 품질(`touchSession` 호출 시점)이 아직 미정이라, 켜면 "0분"같은 거짓 수치로 동기를 깎을 위험. 런타임 부담 때문이 아니라 **데이터 신뢰도** 때문에 미룸.

- **MVP KPI**: 모은 단어 / 모은 문법. (`cards` count — 항상 정확)
- **MVP 책별 진척**: 단어·문법 수 + 진행률(`user_books.progress_pct`), 탭 → 책별 상세.
- **정직 표기**: "읽은 시간·연속일·정답률 통계는 다음 업데이트에서".
- **다음 단계(세션 추적 배선 후)**: 읽은 시간·**연속일(악센트 강조)** KPI / **이번 주 막대**(일별 읽은 시간, 오늘 강조, _안 한 날도 낮은 막대일 뿐 — 빨간 경고 금지, 동기 보호_). (`reading_sessions`)

### 6.7 통계 — 책별 상세 (BookStats)

- **MVP 요약**: 진행률 + 단어/문법 수 + 마지막 읽음(`user_books.last_opened_at`).
- **MVP 챕터별**: 챕터별 카드 수. (`cards.chapter`)
- **액션**: "단어장 보기"(→ 6.3 그 책) / "학습 시작"(→ 6.4). **카드 목록 재나열 금지 — 단어장으로 연결**(중복 방지).
- **다음 단계(세션 추적 후)**: 읽은 시간 / 챕터별 완독 체크·체류 시간. (`reading_sessions`)

### 6.8 상태별 UI (EmptyState / Loading / ErrorState)

- **로딩**(AI 1~3초): 빈 스피너 대신 **카드 골격 스켈레톤** + "풀어보는 중". (멈춤 오해 방지)
- **빈 화면**: 항상 **다음 행동 1개**. 빈 서재 = "첫 책 추가"(=온보딩) / 빈 단어장 = "읽다가 +로 담아보세요" / 빈 통계 = "읽기 시작하면 쌓여요".
- **에러**: 사람 말투 + 재시도. 502/네트워크 = "연결이 잠깐 끊겼어요 · 다시 시도" / **429 = "오늘 AI 사용량을 다 썼어요 · 내일 채워져요(읽기·복습은 계속 가능)"**. 사용자 탓 금지.

### 6.9 가입 유도 (LinkAccountSheet)

- **트리거: 모은 단어가 50개에 도달할 때 1회.** "나중에" = 존중(반복 금지), 더 큰 마일스톤에서 1회 더 가능. **항상 계정 아이콘으로 수동 접근.**
- **프레임: 막기 X, 지키기 O** — "모은 단어 50개, 기기 바뀌어도 안 잃게 저장해둘까요?".
- **폼**: Google / 이메일+비번. **`linkIdentity`로 user_id·데이터 그대로 유지**(plan_v3 인증 모델 2).

## 9. 현재 빌드 범위 vs 다음 단계

- **현재 빌드(이 문서): 6.1 ~ 6.9.** 단, 아래 부분 제외:
  - 6.4 플래시카드는 **포함**하되 **SRS 간격 계산(SM-2)·정답 평가는 다음 단계** (넘기기·뒤집기·범위 시트까지가 현재 빌드. 데이터는 cards에 이미 있어 런타임 가벼움).
  - 6.6/6.7 통계는 **카드 기반만** 현재 빌드, **세션 기반(읽은 시간·연속일·주간 막대)은 다음 단계**.
- **구현 순서(범위 ≠ 시퀀싱)**: 범위에 넣더라도 _만들 때는_ 핵심 루프부터. 핵심 루프(라이브러리→리더→Ask AI→응답 렌더→카드 저장→단어장 목록)를 끝까지 동작시킨 뒤 플래시카드·통계를 얹는다. plan_v3 마일스톤(M1~M8)과 정렬.
- **다음 단계(보류)**: SRS 간격 계산·복습 UI·알림 / 세션 기반 통계 / 문장 컬렉션 UI / 추천(큐레이션) 책 카탈로그 / 다크모드 토큰 적용.
